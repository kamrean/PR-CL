# PR/CI 红灯诊断与 GitHub 仓库体检

## 一句话介绍

我帮你检查 GitHub 仓库的工程化问题，诊断 CI 为什么失败，并帮你落地最基础的 PR/CI 自动化流程，让代码更容易被 review、测试和合并。

## 适合谁

1. 学生项目队：项目能跑，但没有规范 CI。

2. 独立开发者：没人帮忙 review，出错只能自己看日志。

3. 小型外包团队：项目多，构建失败浪费时间。

4. 小 SaaS 团队：已经在 GitHub 上协作，但工程化流程薄弱。

5. 开源维护者：PR 多、issue 多，需要先做初筛。

## 我能帮你解决什么

1. 仓库为什么别人 clone 后跑不起来。

2. GitHub Actions / CI 为什么红灯。

3. PR 里是否有明显风险。

4. 是否缺少测试、lint、build 这些基础检查。

5. 如何把仓库改成更容易协作和交付的状态。

## 交付内容

基础体检包：

- 仓库问题清单
- 风险分级
- 修复优先级
- 下一步建议

CI 红灯诊断：

- 失败步骤
- 关键错误日志
- 可能根因
- 修复建议
- 验证命令

GitHub Actions 落地包：

- 基础 workflow
- test / lint / build 自动运行
- 失败日志说明
- 后续优化建议

## 我不做什么

- 不卖 GPT、Claude、Codex 账号
- 不代登录客户账号
- 不承诺自动修复所有业务代码
- 不直接接生产部署责任
- 不处理未脱敏的敏感数据、密钥、真实用户信息

## 试点报价

CI 红灯单次诊断：

- 试点价：99-299 元/次
- 交付周期：24 小时内

GitHub 仓库体检：

- 试点价：499-999 元/仓库
- 交付周期：1-2 天

基础 GitHub Actions 落地：

- 试点价：1999-4999 元/仓库
- 交付周期：2-3 天

月度 PR/CI 托管：

- 试点价：999-4999 元/月
- 适合：每月有持续 PR 或 CI 问题的小团队

## 客户需要提供什么

1. GitHub 仓库链接。

2. CI 失败链接或日志。

3. 项目使用的语言和框架。

4. 安装依赖和运行测试的命令。

5. 是否允许我查看 PR diff。

6. 是否已对敏感日志做脱敏。

## 首次合作建议

先从一次低风险的仓库体检或 CI 红灯诊断开始。确认报告质量和建议有效后，再升级到 GitHub Actions 落地或月度托管。
