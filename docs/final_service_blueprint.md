# 最终服务蓝图

## 定位

这是一个从人工服务起步、逐步自动化的 PR/CI 工程化服务包。

目标客户：

- 学生项目队
- 独立开发者
- 小型外包团队
- 小 SaaS 团队
- 旧仓库维护者

核心卖点：

把客户仓库从“能跑但不稳定”变成“能体检、能自动检查、CI 红灯能快速定位”。

## 三个产品

### 产品 1：仓库体检包

交付内容：

- 仓库体检报告
- 7 项固定检查
- 风险分级
- 修复优先级
- 下一步服务建议

固定检查项：

- README
- 依赖安装
- 测试命令
- CI 配置
- secrets 风险
- PR 模板
- 分支保护

建议价格：

- 499-999 元/仓库

交付周期：

- 1-2 天

### 产品 2：CI 红灯诊断包

交付内容：

- 失败步骤
- 关键日志
- 根因判断
- 修复建议
- 验证命令
- 风险等级
- 是否适合自动修复

建议价格：

- 99-299 元/次
- 999-2999 元/月小团队托管

交付周期：

- 24 小时内

### 产品 3：PR/CI 自动化落地包

交付内容：

- 基础 GitHub Actions workflow
- Node/Python 最小 CI
- artifact 上传
- workflow 调试说明
- 后续 AI 分析接入建议

建议价格：

- 1999-4999 元/仓库

交付周期：

- 2-3 天

## 标准成交路径

第一步：

先卖低风险的仓库体检包。

第二步：

如果客户有红灯问题，卖 CI 红灯诊断包。

第三步：

如果客户没有基础 CI，卖 PR/CI 自动化落地包。

第四步：

如果客户每月都有 PR 或 CI 问题，转月度托管。

## 不做什么

- 不卖账号
- 不代登录
- 不承诺全自动修业务代码
- 不接生产部署责任
- 不处理未脱敏敏感日志
- 不做公开海外模型入口站

## 关键指标

试点期只看：

- 客户是否愿意付费
- 诊断是否准确
- 报告是否能看懂
- CI 是否能从红灯变绿
- 客户是否愿意继续托管

不要过早追求：

- 完整 SaaS
- 多租户平台
- 自动修复所有问题
- 大规模广告投放
