# Day 2 - 样板仓库评分表

## 使用方法

每个候选样板仓库按 10 个维度打分，每项 0-2 分，总分 20 分。

评分建议：

- 16-20 分：非常适合做 demo
- 12-15 分：可以做 demo，但需要整理
- 8-11 分：只适合内部练习
- 0-7 分：暂不建议使用

## 评分维度

| 维度 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 技术栈常见度 | 冷门 | 一般 | Node/Python/前端等常见栈 |
| 代码规模 | 太大或太小 | 中等但复杂 | 10-100 个文件，易读 |
| 运行方式清晰 | 完全不清楚 | 部分清楚 | README 或配置很清楚 |
| 可制造 CI 错误 | 很难 | 可以但不自然 | 容易制造真实错误 |
| 安全风险 | 有密钥/敏感数据 | 不确定 | 无敏感数据 |
| 展示效果 | 不好展示 | 一般 | 很适合截图和讲故事 |
| 客户代表性 | 很少见 | 有一定代表性 | 高度代表首批客户 |
| 修复边界 | 涉及业务核心 | 中等风险 | 低风险，适合演示 |
| 报告可读性 | 难解释 | 可以解释 | 小白客户也能懂 |
| 复用价值 | 不可复用 | 部分复用 | 可沉淀成模板 |

## Day 2 推荐先选的 3 类

1. 前端项目：优先选择 Vite / React / Vue / Next.js 小项目。

2. Python 项目：优先选择 pytest / requirements.txt / pyproject.toml 项目。

3. 旧仓库：优先选择没有 CI、README 简陋、但代码不敏感的小项目。
