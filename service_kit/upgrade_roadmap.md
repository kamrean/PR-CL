# 能力升级路线

## 阶段一：人工服务型 MVP

目标：

- 用固定模板交付结果
- 用低风险入口包验证需求
- 建立客户信任

当前对应能力：

- 仓库体检
- GitHub Actions 的 CI 红灯诊断
- PR 初审
- 基础 workflow 落地

## 阶段二：半自动执行层

目标：

- 把高重复动作变成固定脚本
- 降低手工整理报告成本

建议优先补：

- CI 红灯报告自动骨架
- PR 初审报告自动骨架
- 标准输入转标准输出

## 阶段三：GitHub 原生自动化

目标：

- 让更多动作在 GitHub 里完成
- 让交付更像产品，而不是一次性服务

建议方向：

- PR 自动评论
- CI 失败后自动生成摘要
- 复跑和验证记录自动沉淀

## 阶段四：托管与扩展

目标：

- 从单次服务走向月度服务
- 从基础检查走向分层能力

建议方向：

- 月度 PR/CI 托管
- GitHub Actions 分层升级
- 前端冒烟测试
- 安全扫描
