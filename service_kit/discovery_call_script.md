# 首次沟通话术

## 目标

第一次沟通不是讲很多，而是快速判断：

1. 客户真正卡在哪
2. 适合卖哪个入口服务
3. 有没有高风险边界

## 开场

推荐说法：

```text
我先不急着讲服务，想先确认你现在最卡的是仓库体检、CI 红灯、PR review，还是 GitHub Actions 落地。
```

## 需要问的 5 个问题

1. 你现在最痛的问题是什么？

2. 这个问题出现在哪个仓库或哪个 PR / CI 链接里？

3. 现在有没有脱敏后的日志或公开仓库链接？

4. 你更想先拿结论，还是希望我直接帮你补基础流程？

5. 如果这次结果有帮助，你更可能继续买单次，还是包月？

## 不要一开始就问的

避免：

- 你预算多少？
- 你要不要做一整套平台？
- 你想不想长期合作？

前期先让客户把具体问题说清楚，更容易成交。

## 如何判断卖哪个服务

如果客户说：

- “仓库很乱，不知道先补什么”
  卖：仓库体检包

- “CI 红灯了，不想看日志”
  卖：CI 红灯单次诊断

- “PR 不知道该不该合”
  卖：PR 初审

- “我们还没有 GitHub Actions”
  卖：GitHub Actions 落地包

## 结束语

推荐说法：

```text
你这个情况我建议别一上来做大包，先从一个低风险入口开始。我更建议先做 [仓库体检 / CI 红灯诊断 / PR 初审]，先把这次问题讲清楚，再决定要不要继续升级。
```
