# 结果展示 SOP

## 目标

结果展示不是把所有报告都发出来，而是把最能体现价值的部分挑出来。

## 适合展示什么

推荐展示：

1. 一句话结论
2. 最重要的 3 个问题
3. 最值得先做的 3 个动作
4. Before / After 对比
5. 一小段脱敏日志
6. 一个风险判断例子

## 不适合展示什么

不要展示：

1. 完整未脱敏日志
2. 客户私有仓库路径
3. token、key、cookie、私钥
4. 真实用户个人信息
5. 客户未授权截图
6. 只有过程没有结果的内容

## 展示顺序

推荐顺序：

1. 客户场景
2. 我们做了什么
3. 结果是什么
4. 为什么有价值

## 最适合用来展示的格式

### 1. GitHub README

适合放简版案例摘要。

### 2. 服务介绍页

适合放 1-3 个代表案例。

### 3. 私信沟通

适合发一段短版案例摘要，不要一开始发完整报告。

### 4. 试点结束后复盘

适合把案例整理成标准化模板，后续复用。

## 展示审核流程

在对外展示前必须确认：

1. 是否已脱敏
2. 是否已获授权
3. 是否只展示必要部分
4. 是否能体现清晰价值
5. 是否容易让客户看懂

## 最后目标

一份好的结果展示，不应该只是“我们很专业”，而应该让客户想到：

这个问题很像我现在的情况，我也想先试一次。
