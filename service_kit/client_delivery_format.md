# Day 6 - 客户交付格式

## CI 红灯诊断交付格式

```text
结论：
这次 CI 失败大概率是因为 [原因]。

失败位置：
[workflow/job/step]

关键日志：
[脱敏后的关键日志]

判断依据：
[为什么认为是这个原因]

建议修复：
1. [建议 1]
2. [建议 2]
3. [建议 3]

验证命令：
[命令]

风险等级：
低 / 中 / 高

是否需要你确认：
是 / 否

下一步：
[建议客户下一步怎么做]
```

## PR Review 交付格式

```text
本次改动摘要：
[一句话说明]

需要注意的问题：
1. [具体问题]
2. [具体问题]
3. [具体问题]

建议补充的测试：
[测试建议]

风险等级：
低 / 中 / 高

是否建议合并：
可以 / 暂缓 / 需要人工确认

建议 reviewer 重点看：
[文件/函数/逻辑]
```

## 交付语气

推荐：

- 直接
- 清楚
- 有证据
- 有下一步

避免：

- 可能是某种复杂问题
- 建议全面检查
- 应该是环境问题
- 你自己看看日志

## 交付前最后检查

1. 客户能不能一眼看懂问题。
2. 客户知不知道下一步做什么。
3. 有没有泄露敏感信息。
4. 有没有过度承诺。
5. 是否明确风险等级。
