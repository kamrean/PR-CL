# 前 50 个潜在客户计划

## 目标

前 50 个触达对象不是为了大面积转化，而是为了验证：

1. 哪种话术回复率最高
2. 哪种服务最容易成交
3. 哪种客户最适合做第一批案例

## 分组建议

### 组 1：熟人和弱关系

数量：

10 人

来源：

- 朋友
- 校友
- 开发群熟人
- 之前合作过的人

目标：

- 拿到 1-2 个试点

### 组 2：公开仓库开发者

数量：

15 人

来源：

- GitHub 上有公开项目的人
- 独立开发者
- 小型开源维护者

目标：

- 拿到 1-2 个公开或匿名案例

### 组 3：学生项目队/社团/实验室

数量：

10 人

来源：

- 学生创业队
- 比赛项目组
- 实验室项目负责人

目标：

- 拿到仓库体检或 GitHub Actions 落地试点

### 组 4：小外包团队/小 SaaS 团队

数量：

15 人

来源：

- 朋友介绍
- 公开官网
- 社交平台开发者账号

目标：

- 拿到 1-2 个付费 CI 或 PR 试点

## 每次触达后要记录

1. 联系对象
2. 所属类型
3. 发送了哪个模板
4. 是否回复
5. 回复关注点是什么
6. 最终是否转化

## 最重要的验证目标

前 50 个触达里，最想验证的是：

1. 体检包是不是最容易成交
2. CI 红灯诊断是不是最容易让客户马上感到值
3. GitHub Actions 落地是不是更适合作为第二单
