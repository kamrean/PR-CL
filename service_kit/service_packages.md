# 服务套餐与定价草案

## 定价原则

我们先卖轻交付、快反馈、低风险的服务，不一开始卖复杂大单。

第一阶段定价看 4 件事：

1. 客户能不能快速听懂
2. 我们能不能快速交付
3. 风险边界清不清楚
4. 是否有升级空间

## 套餐 1：仓库体检包

适合：

- 新项目准备工程化
- 旧仓库准备接手
- 想知道仓库哪里最乱

包含：

- 7 项固定检查
- 总分和健康等级
- 三个最重要问题
- 三个优先修复动作
- 后续建议

建议价格：

- 499-999 元 / 仓库

建议周期：

- 1-2 天

升级方向：

- GitHub Actions 落地
- PR 模板与分支保护补齐

## 套餐 2：CI 红灯单次诊断

适合：

- GitHub Actions 红灯
- build 失败
- test 失败
- lint 失败
- 环境配置失败

包含：

- 失败步骤
- 关键日志
- 根因判断
- 修复建议
- 验证命令
- 风险等级

建议价格：

- 99-299 元 / 次

建议周期：

- 24 小时内

升级方向：

- 创建修复 PR
- 月度 CI 托管

## 套餐 3：PR 初审包

适合：

- 没有固定 reviewer
- PR 改动不大，但怕合进去出问题
- 想快速做一轮风险筛查

包含：

- 改动摘要
- 潜在 bug
- 缺失测试
- 兼容性风险
- 安全或权限风险
- 是否建议合并

建议价格：

- 99-399 元 / PR

建议周期：

- 24 小时内

升级方向：

- 月度 PR 初审托管

## 套餐 4：GitHub Actions 落地包

适合：

- 还没有基础 CI
- 有 CI，但很乱
- 想让 PR 自动检查

包含：

- 基础 workflow
- Node 或 Python 最小 CI
- artifact 上传
- 调试清单
- 后续优化建议

建议价格：

- 1999-4999 元 / 仓库

建议周期：

- 2-3 天

升级方向：

- 月度 PR/CI 托管
- 自动诊断接入

## 套餐 5：月度 PR/CI 托管

适合：

- 每月持续有 CI 红灯
- PR 数量稳定
- 需要轻量工程化支持

包含：

- 固定次数的 CI 红灯诊断
- 固定次数的 PR 初审
- 仓库体检复查
- 小型 workflow 优化建议

建议价格：

- 999-4999 元 / 月

建议周期：

- 按月服务

## 推荐成交路径

最适合的销售顺序：

1. 先卖 `仓库体检包`
2. 再卖 `CI 红灯单次诊断`
3. 再卖 `GitHub Actions 落地包`
4. 最后转 `月度 PR/CI 托管`

## 哪些情况要单独报价

以下情况不要套标准价：

- 私有仓库且上下文复杂
- 依赖生产环境
- 涉及登录、支付、权限、数据库
- 需要连续多轮修复
- 需要额外文档或培训

## 报价话术

推荐说法：

这次先建议从低风险入口包开始，比如仓库体检或 CI 单次诊断。你先看报告质量和建议是否有用，再决定要不要继续升级自动化落地。

避免说法：

- 我们可以全自动帮你搞定
- 什么都能修
- 保证一次解决所有问题
