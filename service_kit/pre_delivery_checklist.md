# 交付前检查表

## 使用目标

在把任何报告、建议、模板发给客户之前，先过这一遍检查，避免把低质量结果交出去。

## 1. 交付内容是否完整

- 是否有一句话结论
- 是否有最重要的 3 个问题
- 是否有下一步建议
- 是否有详细报告
- 是否有风险等级
- 是否有是否需要客户确认

## 2. 证据是否充分

- 结论是否有日志或仓库证据支持
- 是否引用了具体文件/步骤/命令
- 是否区分了“确定事实”和“可能原因”
- 是否避免了空话

## 3. 安全与隐私

- 是否包含 token
- 是否包含 API key
- 是否包含 cookie
- 是否包含 private key
- 是否包含真实用户个人信息
- 是否包含生产数据库地址
- 是否包含未脱敏截图

## 4. 可执行性

- 客户是否知道下一步做什么
- 是否给了具体命令
- 是否给了具体文件或配置位置
- 是否写明优先级

## 5. 风险边界

- 是否涉及登录/支付/权限
- 是否涉及数据库
- 是否涉及生产部署
- 是否涉及 workflow 权限扩大
- 是否明确标为高风险

## 6. 商业交付质量

- 客户能不能 3 分钟内看懂重点
- 是否能体现这项服务的价值
- 是否适合作为案例沉淀
- 是否留了升级服务的入口

## 通过标准

只有全部满足以下条件，才允许发给客户：

1. 没有敏感信息
2. 有清楚结论
3. 有可执行建议
4. 有风险标记
5. 有下一步动作

## 不通过时怎么处理

如果不通过：

1. 先不发客户
2. 回到模板补内容
3. 缩小范围
4. 补充输入材料
5. 再复核一次
