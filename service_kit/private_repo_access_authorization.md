## 目标

这份模板用于记录和确认私有仓库访问边界，避免出现“先把权限给我，做完再说”的模糊合作。

## 访问方式

请只选择一种主方式：

- [ ] 只读协作者
- [ ] 临时写权限协作者
- [ ] GitHub App
- [ ] Fine-grained PAT
- [ ] 客户导出脱敏资料，不提供仓库访问

## 最小权限原则

默认优先顺序：

1. 无仓库访问，只看脱敏资料
2. 只读访问
3. 临时 PR 创建权限
4. 临时 workflow 修改权限

以下权限默认不申请：

- 管理员权限
- 永久写权限
- 生产环境 secrets 管理权限
- branch protection 管理权限
- environment 管理权限

## 本次允许的动作

- [ ] 查看仓库代码

- [ ] 查看 GitHub Actions 结果

- [ ] 修改 `.github/workflows/*`
- [ ] 重新触发 CI

## 本次明确禁止的动作

- [ ] 直接 push 到默认分支
- [ ] 修改生产环境 secrets
- [ ] 修改 branch protection
- [ ] 修改 deployment environment
- [ ] 修改支付、登录、权限控制逻辑
