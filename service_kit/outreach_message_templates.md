# 首批客户私信模板

## 使用原则

第一批私信不是为了“广撒网推销”，而是为了找最容易成交的试点客户。

建议优先联系：

1. 学生项目队
2. 独立开发者
3. 小型外包团队
4. 有公开 GitHub 仓库的小 SaaS 团队
5. 你已经认识的人

不要一开始就联系：

1. 大型企业正式采购团队
2. 高敏感行业生产系统负责人
3. 明显需要重度 DevOps 的客户

## 模板 1：仓库体检邀约

适合：

- 看到对方仓库 README 很乱
- 没有 GitHub Actions
- 仓库明显缺工程化基础

文案：

```text
我最近在做一个 GitHub 仓库工程化体检服务，主要帮人快速看 README、测试命令、CI、PR 模板、secrets 风险这些基础问题。

我看了下你的仓库，感觉很适合做一个低成本体检样例。我可以先给你出一版简短体检报告，重点告诉你：

1. 现在最影响协作的 3 个问题
2. 最值得先补的 3 个动作
3. 是否适合接 GitHub Actions 和 PR 自动检查

如果你愿意，我可以先按试点价做，或者先免费做一个简版样例换反馈。
```

## 模板 2：CI 红灯诊断邀约

适合：

- 对方经常发 CI 失败截图
- 对方在群里问构建为什么挂了
- 对方仓库最近有红灯

文案：

```text
我最近在做一个专门帮人看 GitHub Actions / CI 红灯的服务。

如果你现在正好有一条构建失败日志，我可以帮你把它整理成一份可执行诊断：

1. 失败发生在哪一步
2. 最可能的根因
3. 修复建议
4. 你下一步应该先跑什么命令

这个服务比较适合“不想自己啃长日志，但又不想随便乱改”的情况。你如果愿意，可以发我一段脱敏后的关键日志，我先给你看下值不值得继续做。
```

## 模板 3：PR 初审邀约

适合：

- 对方有公开 PR
- 对方缺固定 reviewer
- 小团队自己 review 压力大

文案：

```text
我最近在做一个轻量 PR 初审服务，不是替代正式 code review，而是先帮你把明显问题筛一轮：

1. 改动摘要
2. 潜在 bug
3. 缺失测试
4. 兼容性/安全风险
5. 是否建议先别合并

如果你最近有一个想快速过一轮的 PR，我可以先按试点价给你做一个样例。
```

## 模板 4：GitHub Actions 落地邀约

适合：

- 对方还没有 CI
- 对方每次都靠人工运行测试
- 对方仓库适合先跑基础 build/test

文案：

```text
我最近在做 GitHub Actions 基础落地服务，适合那种“代码能写，但工程化还没补起来”的仓库。

第一版不会做很复杂，只会先帮你把：

1. install
2. test
3. lint
4. build

这些最基础检查接到 PR 流程里。这样以后每次开 PR，系统就能先帮你过一轮自动检查。

如果你有一个仓库正好还没 CI，我可以按试点价先做一个最小版本。
```

## 模板 5：熟人转介绍

适合：

- 朋友
- 校友
- 开发群里熟人
- 老客户转介绍

文案：

```text
我现在在做一个偏 GitHub 工程化的小服务，主要是：

1. 仓库体检
2. CI 红灯诊断
3. PR 初审
4. GitHub Actions 基础落地

如果你身边有下面这几类人，可以帮我转一下：

- 仓库很乱但不知道先补什么的人
- GitHub Actions 老是红灯的人
- 没时间认真看 PR 的小团队
- 想补基础 CI 但没人动手的人

我现在主要接试点单，价格会低一些，重点是先做案例和口碑。
```
