# 案例授权模板

## 使用目标

试点和正式交付后，如果客户愿意让你展示案例，最好提前把授权边界说清楚。

## 模板

你可以把下面这段发给客户：

```text
这次合作如果你觉得有帮助，我希望能把它整理成一个匿名案例，用于展示我的服务能力。

我默认遵守这些边界：

1. 不公开仓库私密信息
2. 不公开 token、key、cookie、私钥、环境变量
3. 不公开真实用户个人信息
4. 不公开你不允许展示的截图或日志
5. 默认只展示问题类型、分析过程、结果形式和可公开的前后对比

如果你愿意，请确认以下选项：

- [ ] 允许匿名展示这个案例
- [ ] 允许展示脱敏后的日志片段
- [ ] 允许展示 workflow 或模板片段
- [ ] 允许展示问题类型和修复结果
- [ ] 需要先给你看一版展示稿
```

客户名称：

是否允许匿名展示：

是否允许展示截图：

是否允许展示 workflow 片段：

是否需要客户预览后再公开：

备注：

## 最重要的原则

1. 不要默认客户允许展示。

2. 不要把“匿名”理解成“可以随便展示”。

3. 任何涉及私有仓库、内部系统、敏感日志的内容，都先让客户确认。
