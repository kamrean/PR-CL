# 交付前安全硬检查

## 目标

这份清单是每次正式交付前必须走的一次硬检查。

## 1. 资料来源

- [ ] 公开仓库只收公开信息
- [ ] 私有资料走私下渠道
- [ ] 没有把客户真实日志贴进公开 issue

## 2. 敏感信息

- [ ] 没有 token
- [ ] 没有 key
- [ ] 没有 cookie
- [ ] 没有 private key
- [ ] 没有数据库连接串
- [ ] 没有客户个人身份信息

## 3. 权限

- [ ] 仓库访问方式已记录
- [ ] 最小权限边界已确认
- [ ] 是否允许建 PR 已确认
- [ ] 是否允许改 workflow 已确认
- [ ] 撤权责任人已确认

## 4. AI 外发

- [ ] 外发内容已脱敏
- [ ] 客户已知情或已授权
- [ ] 高风险内容未外发
- [ ] AI 输出已人工复核

## 5. Artifact

- [ ] 只上传白名单内文件
- [ ] 未上传截图和构建目录
- [ ] retention 已设置
- [ ] 客户知道 artifact 会保留多久

## 6. 案例展示

- [ ] 仅展示已授权内容
- [ ] 文件名不暴露客户身份
- [ ] 客户要求匿名时已匿名
- [ ] 客户要求不展示时已排除

## 7. 交付包

- [ ] 命名不含真实客户敏感信息
- [ ] 报告里没有未脱敏日志
- [ ] 风险边界写清楚
- [ ] 下一步建议没有越权承诺
