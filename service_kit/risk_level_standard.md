# 风险分级标准

## 低风险

典型场景：

- README 缺失
- 命令说明不清楚
- workflow 漏基础步骤
- lint / format 问题
- PR 模板缺失

处理方式：

- 可以直接给建议
- 可以在客户确认后落地

## 中风险

典型场景：

- 依赖升级
- 测试断言调整
- 环境变量配置
- 构建配置修改
- workflow 权限小幅调整

处理方式：

- 先说明影响范围
- 需要客户确认后推进

## 高风险

典型场景：

- secrets 泄露
- 支付、登录、权限相关改动
- 数据库迁移
- 生产部署
- 真实用户数据

处理方式：

- 默认停止继续推进
- 必须单独确认边界和责任
