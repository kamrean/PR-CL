# PR/CI 服务边界

## 目标

这份文档用来明确：

- 我们做什么
- 我们暂时不做什么
- 哪些内容必须人工确认

## 当前提供的服务

1. 仓库体检
2. CI 红灯诊断
3. PR 初审
4. GitHub Actions 基础落地

## 当前默认支持范围

- GitHub
- GitHub Actions
- Node 项目
- Python 项目

## 默认不做

- 账号共享
- 代登录
- 海外模型访问转售
- 无边界的全自动修复
- 生产部署托管
- 未脱敏敏感资料处理

## 必须人工确认的内容

- 登录
- 支付
- 权限控制
- 数据库迁移
- secrets
- 生产配置
- 任何可能改动业务逻辑的修复

## 适合先推进的低风险内容

- README 和文档补齐
- 基础 workflow
- lint / test / build
- 低风险 CI 红灯定位
- PR 风险初筛

## 对客户的默认承诺

- 先做低风险入口包
- 先给出结构化结论
- 高风险改动先停下来确认
- 不把未脱敏内容发给 AI
