# 客户常见问题 FAQ

## 1. 你们和卖 GPT / Claude 账号有什么区别？

完全不同。

我们不卖账号、不代登录、不共享模型访问资格。我们卖的是：

- 仓库体检结果
- CI 红灯诊断结果
- PR 初审结果
- GitHub Actions 自动化落地结果

## 2. 你们会直接改我的业务代码吗？

第一阶段不会默认直接改业务代码。

我们先给结论、风险和修复建议。只有在低风险场景，且你明确允许时，才会继续进入修复 PR 或 workflow 修改。

## 3. 什么样的问题最适合先找你们？

最适合的是这几种：

- GitHub 仓库看起来乱，不知道先补哪块
- GitHub Actions 红灯，不想自己啃长日志
- PR 想快速做一轮风险筛查
- 没有基础 CI，想先让仓库自动检查起来

## 4. 你们能保证 CI 一定变绿吗？

不能做这种绝对承诺。

我们能承诺的是：

- 尽量快速定位问题
- 给出可执行建议
- 明确风险边界
- 帮你区分能快速修和必须谨慎处理的问题

## 5. 你们会接触我的密钥吗？

默认不接触。

如果日志、配置或 workflow 里出现 token、key、cookie、private key，我们会先提醒你脱敏，必要时建议立即轮换密钥。

## 6. 你们适合什么样的团队？

最适合：

- 学生团队
- 独立开发者
- 小型外包团队
- 小 SaaS 团队
- 旧仓库维护者

不太适合一上来就找我们做的：

- 大型企业核心生产系统
- 高度敏感的金融/医疗/政务生产环境
- 需要重度 DevOps 值班的团队

## 7. 一次合作通常怎么开始？

最常见的开始方式：

1. 发仓库链接做体检
2. 发一段脱敏后的 CI 红灯日志做单次诊断

## 8. 报告长什么样？

报告不会只有“几句建议”，通常至少包含：

- 一句话结论
- 最重要的 3 个问题
- 修复优先级
- 详细报告
- 风险确认
- 下一步建议

## 9. 你们会不会直接把 AI 输出原样发给我？

不会。

AI 只是辅助分析，最终交付前会有人做人工复核，避免：

- 误判
- 编造
- 漏看关键 error
- 泄露敏感信息

## 10. 如果第一次合作，最建议买哪个服务？

最建议从：

- 仓库体检包
或
- CI 红灯单次诊断

开始。

这两个成本低、风险低、最容易看出是否值得继续合作。
