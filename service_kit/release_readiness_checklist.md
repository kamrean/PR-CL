# 上线前总检查清单

## 目标

这份清单用于在正式对外展示、发给客户、或者 push 到 GitHub 首页之前，做最后一次全局检查。

## 1. 仓库结构检查

- [ ] 根目录只保留必要入口文件
- [ ] 所有服务材料都在 `service_kit/`
- [ ] workflow 模板在 `workflow_templates/`
- [ ] `.github/` 目录可用
- [ ] README 导航没有失效链接

## 2. 服务完整性检查

- [ ] 仓库体检材料完整
- [ ] CI 红灯诊断材料完整
- [ ] PR 初审材料完整
- [ ] GitHub Actions 落地材料完整
- [ ] AI 分析与人工复核材料完整
- [ ] 交付包材料完整

## 3. 销售与获客检查

- [ ] 服务主页文案存在
- [ ] 套餐与定价存在
- [ ] 客户 FAQ 存在
- [ ] 首批客户私信模板存在
- [ ] 试点邀约文案存在
- [ ] 客户筛选评分存在

## 4. 试点与跟进检查

- [ ] 试点执行包存在
- [ ] 试点验收标准存在
- [ ] 试点转付费 SOP 存在
- [ ] 客户触达记录表存在
- [ ] 试点反馈表存在
- [ ] 线索跟进 SOP 存在

## 5. 案例展示检查

- [ ] 案例展示页存在
- [ ] 案例摘要模板存在
- [ ] Before / After 模板存在
- [ ] 结果展示 SOP 存在
- [ ] 展示素材检查表存在

## 6. 安全与合规检查

- [ ] 仓库里没有 `.env`
- [ ] 仓库里没有 token / key / cookie / 私钥
- [ ] 没有未脱敏客户日志
- [ ] 没有真实用户个人信息
- [ ] 没有生产数据库地址
- [ ] 案例展示内容都有授权或匿名化

## 7. GitHub 展示检查

- [ ] README 首页清楚说明项目定位
- [ ] README 有“先看这里”
- [ ] README 有“服务包”“获客与试点”“案例展示”入口
- [ ] Issue 模板可用
- [ ] PR 模板可用
- [ ] docs-check workflow 可用

## 8. 最终判断

如果下面 5 项都满足，就可以进入公开展示或开始大规模触达：

- [ ] 仓库结构清楚
- [ ] 服务边界清楚
- [ ] 交付模板完整
- [ ] 获客流程完整
- [ ] 没有明显安全问题
