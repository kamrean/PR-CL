# 销售用案例摘要模板

## 使用目标

这份模板适合放在：

- GitHub README
- 对外介绍页
- 私信回复
- 销售文案

它不是完整报告，而是 30 秒内讲清楚案例价值。

## 模板

### 案例名称

例如：

- 前端 CI 红灯诊断
- 旧仓库工程化体检
- PR 初审与合并建议

### 客户场景

一句话说明客户当时卡在哪里。

例如：

一个前端项目在 GitHub Actions 里 build 失败，团队不知道是路径问题还是配置问题。

### 我们做了什么

最多写 3 条动作：

1.
2.
3.

### 结果

最多写 3 条结果：

1.
2.
3.

### 为什么这个案例有价值

一句话说明价值。

例如：

这个案例的价值不只是定位红灯，而是让客户从“看不懂日志”变成“知道下一步做什么”。

## 短版模板

如果只允许写 3 行，可以用：

```text
客户场景：
[一句话]

我们做了什么：
[一句话]

结果：
[一句话]
```
