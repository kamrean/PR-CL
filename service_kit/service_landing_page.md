# PR/CI 红灯诊所

让你的 GitHub 仓库更容易协作、让 CI 红灯更容易定位、让 PR 更容易通过。

## 我们解决什么问题

很多项目不是“代码完全写不出来”，而是卡在这些地方：

- 仓库说明不清楚，新人接手慢
- CI 经常红灯，但没人愿意看长日志
- PR 改动看起来不大，但 reviewer 很难快速判断风险
- GitHub Actions 没配好，代码问题总是很晚才暴露
- 团队知道要工程化，但不知道先补哪一块

这项服务就是为这些问题准备的。

## 我们卖的不是账号，是结果

我们不卖 GPT、Claude、Codex 账号，不做代登录，不做共享号。

我们真正提供的是：

1. 仓库体检

帮你检查 README、依赖安装、测试命令、CI 配置、secrets 风险、PR 模板、分支保护，给出清晰报告和优先级建议。

2. CI 红灯诊断

帮你从失败日志里提取关键错误，判断最可能根因，给出修复建议和验证命令。

3. PR 初审

帮你快速看一轮改动，指出潜在 bug、缺失测试、兼容性风险和高风险修改。

4. PR/CI 自动化落地

帮你把 GitHub Actions 基础流程补上，让仓库从“靠人肉检查”升级到“每次 PR 自动检查”。

## 适合谁

最适合这几类客户：

1. 学生项目队

代码能写，但 GitHub Actions、CI、PR 流程不稳定。

2. 独立开发者

没有固定 reviewer，CI 一红就只能自己熬夜看日志。

3. 小型外包团队

项目多、交付快，最怕构建失败和仓库接手混乱。

4. 小 SaaS 团队

已经在 GitHub 上协作，但工程化基础还不够扎实。

5. 旧仓库维护者

项目能跑，但没有 README、没有 CI、没有模板、没有保护。

## 你会拿到什么

每次交付都不是“几句建议”，而是完整交付包：

- 一句话结论
- 最重要的 3 个问题
- 建议优先做的 3 个动作
- 详细报告
- 风险确认
- 下一步建议

如果是自动化落地，还会给你：

- GitHub Actions workflow
- 调试清单
- 后续优化建议

## 为什么这项服务对小白也友好

你不需要一开始就会做完整 DevOps，也不需要自己搭复杂平台。

你可以先从最简单的结果买起：

- 看一眼仓库哪里最乱
- 帮我解释 CI 为什么红
- 帮我把基础 GitHub Actions 跑起来

这比“做一个大而全平台”更容易真正解决问题，也更容易控制风险。

## 最适合的第一单

如果你第一次合作，不建议一上来就做复杂自动化。

推荐顺序：

1. 先做仓库体检
2. 再做 CI 红灯诊断
3. 再做 GitHub Actions 落地
4. 最后再考虑月度托管

## 合作边界

我们会明确区分低风险、中风险、高风险问题。

可以快速推进的：

- 文档
- workflow
- lint / test / build
- 低风险 CI 红灯

必须人工确认的：

- 登录
- 支付
- 权限
- 数据库
- secrets
- 生产部署

## 下一步怎么开始

最简单的开始方式只有两种：

1. 发一个仓库链接，做仓库体检
2. 发一段脱敏后的 CI 红灯日志，做单次诊断

如果你愿意继续，我们再把它升级成自动化交付。
