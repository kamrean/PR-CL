# PR/CI 红灯诊所

## 我们解决什么问题

很多项目不是“代码完全写不出来”，而是卡在这些地方：

- 仓库说明不清楚，新人接手慢
- GitHub Actions 经常红灯，但没人愿意看长日志
- PR 改动不大，但 reviewer 很难快速判断风险
- 工程化基础薄弱，问题总是在很晚才暴露

## 我们卖的是什么

我们卖的不是账号，不是共享号，也不是“什么都能自动修”的空承诺。

我们卖的是 4 类可交付结果：

1. 仓库体检
2. GitHub Actions 的 CI 红灯诊断
3. PR 初审
4. GitHub Actions 基础落地

## 当前默认支持范围

当前主支持：

- GitHub
- GitHub Actions
- Node
- Python

其他平台或更重的交付场景，默认按单评估。

## 适合谁

1. 学生项目组
2. 独立开发者
3. 小型外包团队
4. 小 SaaS 团队
5. 旧仓库维护者

## 怎么开始

最简单的开始方式只有两种：

1. 发一个仓库链接，先做仓库体检
2. 发一段脱敏后的 GitHub Actions 错误摘要，先做单次诊断

## 安全边界

公开仓库不是客户私有资料的默认入口。

如果涉及私有仓库、私有 PR、原始日志、截图或 workflow 文件，先走私下渠道，再按权限边界推进。

## 为什么这条路适合先做

因为它的价值最容易被客户感知：

- 红灯有没有变绿
- 报告有没有看懂
- PR 风险有没有提前发现
- workflow 有没有真正跑起来
