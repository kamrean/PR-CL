# 线索跟进 SOP

## 使用目标

跟进的目标不是“多催几次”，而是：

1. 在不打扰的前提下推进成交
2. 快速识别值得继续花时间的人
3. 及时放弃无效线索

## 跟进节奏建议

### 第一次联系

发送对应场景的私信模板。

目标：

- 判断对方是否有兴趣
- 判断问题是否真实存在

### 第一次跟进

时间：

首次联系后 2-3 天

话术：

```text
前两天给你发过一条关于 [仓库体检 / CI 红灯诊断 / PR 初审] 的消息，不确定你有没有看到。

如果你现在刚好有一个仓库或 CI 问题，我可以先帮你看一个低风险样例，先确认这种服务对你有没有价值。
```

### 第二次跟进

时间：

第一次跟进后 4-7 天

话术：

```text
我这边最近在整理第一批试点案例，如果你那边还需要，我可以给你保留一个试点名额。你如果愿意，直接发我仓库链接或一段脱敏后的 CI 日志就行。
```

### 停止跟进

如果对方：

- 连续两次不回复
- 没有真实问题
- 只想白嫖咨询
- 风险明显过高

就标记为 `暂不跟进`。

## 跟进时不要做的事

1. 不要长篇解释自己有多厉害。

2. 不要一次发太多文件。

3. 不要催着对方立刻下单。

4. 不要在没有脱敏的情况下要求对方发完整日志。

5. 不要对高风险项目做过度承诺。

## 跟进后要更新的内容

每次跟进后，记录：

1. 这次用了哪句话术
2. 对方有没有回复
3. 对方最关注的点是什么
4. 是价格问题、信任问题还是需求不明确
5. 下一步该继续还是停止

## 目标结果

一个好的跟进，不一定马上成交，但至少会把线索推进到下面几种状态之一：

- 明确有需求
- 明确无需求
- 愿意试点
- 需要稍后再联系
- 不适合继续跟进
