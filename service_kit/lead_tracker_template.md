# 客户触达记录表模板

## 使用目标

这份表用来记录：

1. 你联系了谁
2. 用了哪条话术
3. 对方有没有回复
4. 目前推进到哪一步
5. 是否值得继续跟进

## 建议字段

| 编号 | 日期 | 姓名/团队 | 类型 | 联系方式 | 仓库/项目 | 当前痛点 | 使用话术 | 是否回复 | 当前状态 | 下次跟进时间 | 备注 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| 1 |  |  | 学生/独立开发者/外包/小团队 |  |  |  | 仓库体检/CI 红灯/PR 初审/Actions 落地 | 是/否 | 未回复/已回复/试点中/已成交/不跟进 |  |  |

## 当前状态说明

建议统一用下面几种：

- 未联系
- 已联系未回复
- 已回复待沟通
- 已确认试点
- 试点进行中
- 已交付待反馈
- 已成交
- 暂不跟进

## 每次跟进后要更新什么

1. 对方最真实的痛点
2. 对方最关心的是价格、速度还是风险
3. 对方更适合哪个入口服务
4. 是否愿意提供仓库/日志
5. 是否愿意以后做案例

## 最重要的记录原则

不要只记“联系过”。要记：

- 对方为什么不回
- 对方为什么有兴趣
- 哪句话最有效
- 哪类客户最容易进试点

## 推荐最小记录

如果你不想一开始做复杂表格，最少也要记录这 6 个字段：

1. 联系对象
2. 当前痛点
3. 发了哪条文案
4. 是否回复
5. 当前状态
6. 下一次什么时候跟进
