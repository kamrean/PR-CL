# 仓库体检检查清单

## 目标

用固定检查项快速判断一个仓库是否具备基本协作和持续交付条件。

## 固定检查项

1. README
2. 依赖安装
3. 测试命令
4. CI 配置
5. secrets 风险
6. PR 模板
7. 分支保护

## README

检查：

- 是否说明项目用途
- 是否写清安装命令
- 是否写清运行命令
- 是否写清测试方式
- 是否说明环境变量要求

## 依赖安装

检查：

- 是否存在清晰的依赖文件
- 安装命令是否能从 README 找到
- 锁文件是否存在
- 依赖安装是否容易复现

## 测试命令

检查：

- 是否存在标准测试命令
- 测试命令是否写进 README
- 新人是否能直接跑起来

## CI 配置

检查：

- 是否已有基础 CI
- 是否在 PR 或 push 时自动触发
- 是否能跑核心检查
- 失败时是否容易定位

## secrets 风险

检查：

- 是否疑似把 secrets 写进仓库
- 是否有可疑配置文件
- 是否把敏感值暴露在文档或样例里

## PR 模板

检查：

- 是否有 PR 模板
- 是否要求说明改动摘要
- 是否要求说明验证方式

## 分支保护

检查：

- 默认分支是否受保护
- 是否禁止直接 push
- 是否要求 PR 合并前通过检查

## 输出要求

每次体检至少输出：

1. 总体判断
2. 最重要的 3 个问题
3. 建议优先做的 3 个动作
4. 风险等级
