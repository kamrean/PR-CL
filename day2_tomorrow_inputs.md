# Day 2 - 明天 Day 3 需要的输入

Day 3 要做的是仓库体检模板。为了让 Day 3 能直接开始，今天需要准备以下输入。

## 必备输入

1. 一个前端样板仓库。

最理想：

- Vite / React / Vue / Next.js
- 有 package.json
- 有 test / lint / build 中至少一个命令

2. 一个 Python 样板仓库。

最理想：

- 有 requirements.txt 或 pyproject.toml
- 有 pytest 测试
- 可以制造一个失败测试

3. 一个旧仓库样板。

最理想：

- README 不完整
- 没有 GitHub Actions
- 没有 PR 模板
- 没有明确测试说明

## 如果今天没有真实仓库

也可以先用假设案例推进：

1. 前端案例：React/Vite 项目，CI build 失败。

2. Python 案例：pytest 失败，requirements 缺依赖。

3. 旧仓库案例：README 简陋，没有 CI 和 PR 模板。

## Day 3 会产出的内容

1. 仓库体检检查清单。

2. 仓库体检报告模板。

3. 仓库风险分级标准。

4. 第一个 demo 仓库体检报告草稿。

## 今日人工动作

如果你能提供真实仓库，优先选：

1. 你自己的旧项目。

2. 朋友的学生项目。

3. 一个小型开源项目。

如果暂时没有真实仓库，就先用这 3 个假设案例继续走完流程，不影响 Day 3。
