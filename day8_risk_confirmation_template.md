### 需要客户确认的事项

- [ ] 是否允许修改依赖版本
- [ ] 是否允许修改 workflow
- [ ] 是否允许创建修复 PR
- [ ] 是否允许补测试或修改测试断言
- [ ] 是否涉及 secrets / token / key
- [ ] 是否涉及环境变量
- [ ] 是否涉及生产部署
- [ ] 是否涉及数据库变更
- [ ] 是否涉及登录、支付、权限逻辑

### 沟通建议

对客户的说法建议是：

1. 先说明问题是什么。

2. 再说明为什么这是风险。

3. 再说明如果你拍板，我们可以继续怎么做。

避免：

- 直接让客户自己处理
- 模糊说有风险但不解释
- 没有下一步建议
